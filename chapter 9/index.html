1. what is Error handling in JS?
Error handling is a mechanism in JavaScript to handle runtime errors and prevent the program from crashing. It involves identifying and handling errors gracefully, allowing the program to continue running and providing useful feedback to the user.


2. what is the role of finally block in JS?
The finally block is a block of code that always executes, whether an error occurred or not.
It is used to perform cleanup operations, such as closing database connections or releasing resources.


3. what is the purpose of throw statement in JS?
The throw statement is used to manually throw an error. It can be used to terminate the execution of a function and pass control to the nearest enclosing try...catch block.


4. what is Error propagation in JS?
Ans:- Error propagation refers to the process of passing or propagating an error from one part of the code to another by using the throw statement with try and catch.


5. What are the best practices for error handling?
Best practices for error handling in JavaScript include:
a. Use descriptive error messages
b. Handle errors gracefully
c. Log errors for debugging purposes
d. Use try...catch blocks
e. Use finally blocks for cleanup operations
f. Throw specific error types for different error scenarios
g. Use Promises and async/await for handling asynchronous operations
h. Use error boundaries to catch errors in nested components
i. Use error boundaries to catch errors in server-side rendering
j. Use try...finally blocks for cleanup operations


6. What are the different types of errors in JS?
Types of errors in JavaScript include:
a. Syntax errors
b. Logical errors
c. Type errors
d. Reference errors
e. Range errors
